InStorage

InStorage is a web application aimed at solving the storage issues of college students, or what we like to say Airbnb for storage combined with Uber for moving! We used React.js to create the app, along with Firebase for our authentication and database needs. We also hooked up an Express backend to deal with any interactions betwen the client and the Firebase database. We performed frontend and backend testing using Jest and Enzyme.

Readible

Readible is a simple article-sharing application that allows users to upload and share their own articles through Google Docs. This is done by publishing your Google Doc and submitting the embedded link to the application. Firebase is used to store the author's name, article title, and the embedded iframe src. React Router is used to route between the different pages and articles. Users can browse through the available articles on the landing page and read more if interested.

Billiards Game

An 8-ball pool game built using Javascript and the tiny-graphics library. Javascript was used to flesh out all of the game logic and user interface features. The tiny-graphics library was used to help model and render the pool table, the cue stick, and the billiard balls.

TuneSearch

TuneSearch is a simple search engine for song lyrics. Given a set of songs and other information, I loaded the data into Postgres and used it to query for key words typed in by the user. Each result lists the name of the song and artists, ranked accordingly to its TF-IDF score. Python was used to connect to the PostgreSQL instance that was running on a VM.

Space Blaster

This is a spinoff of the classic game Astro Blaster. A header file contained all of the game objects that are derived from a single actor base class. Each game object was implemented with different properties, such as flying speed, direction pattern, health, amount of damage, etc. A game world class was used to keep track of all objects in the game, including the player, aliens, moving background stars, collectables, game score, and player lives.

Cracked

Cracked is a decoder for trivially encrypted messages. A tokenizer is used to separate each word of the message and then placed in a vector. Given a large data set of English words, I cross-referenced each encrypted word (starting with the largest) against words that followed the same pattern. The translator class is then updated with the pending mapping and other ensuing mappings are also pushed onto the stack. Any conflicting mappings would pop the current mapping off the stack, and using DFS, a new correct mapping would be found and another pathway taken.
